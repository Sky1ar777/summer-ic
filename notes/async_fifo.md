> 2026/6/10 · Week2 · 面试必考手撕代码

## 1. 为什么需要异步 FIFO？

### 问题：跨时钟域传多位数据

```
写时钟域（100MHz） ─── 8位数据 ──→ 读时钟域（50MHz）
```

如果用**打两拍**方案把 8 位各自同步：

```
数据变化: 00001111 → 11110000
  bit0: 1 ──→ 同步器 ──→ 1     ✅ 立即可见
  bit1: 1 ──→ 同步器 ──→ 1     ✅
  bit2: 1 ──→ 同步器 ──→ 1     ✅
  bit3: 1 ──→ 同步器 ──→ 1     ✅
  bit4: 0 ──→ 同步器 ──→ 0     ✅
  bit5: 0 ──→ 同步器 ──→ 1     ❌ 走线稍长，比其它bit晚1拍
  bit6: 0 ──→ 同步器 ──→ 1     ❌ 更晚
  bit7: 1 ──→ 同步器 ──→ 1     ✅

读时钟域某时刻看到: 11111111（全是1！）❌❌❌
```

**结论：不能对多位总线各自打拍！**

### 解决方案

FIFO（First In First Out）：
- 写时钟域只管往 FIFO **写**数据
- 读时钟域只管从 FIFO **读**数据
- 中间用**双端口 RAM** 隔离

问题转化为：**怎么知道 FIFO 是空还是满？**

## 2. 核心设计：空满判断

### 基本结构

```
写时钟域                      读时钟域
┌──────────┐                ┌──────────┐
│  waddr   │───→ 双端口 RAM ←───│  raddr   │
│  wptr    │                │  rptr    │
└────┬─────┘                └────┬─────┘
     │                           │
     ↓ 同步到读时钟域              ↓ 同步到写时钟域
   sync_rptr                    sync_wptr
```

### 空满判断原理

```
空：读指针 == 写指针
满：写指针超前读指针一个FIFO深度
```

但比较的时候需要跨时钟域——**用格雷码！**

## 3. 格雷码（Gray Code）

### 为什么要用格雷码？

格雷码特性：**每步只变 1 位**

```
二进制: 000 → 001 → 010 → 011 → 100 → 101 → 110 → 111
格雷码: 000 → 001 → 011 → 010 → 110 → 111 → 101 → 100
                         ↑ 每次只变1bit！
```

**格雷码 + 打两拍** = 即使同步有时序偏差，要么读到旧值要么读新值，绝不会读到"中间值"。

### 二进制→格雷码

```
gray = (bin >> 1) ^ bin
```

举例：
```
bin=101 (5)：gray = (010) ^ 101 = 111
bin=110 (6)：gray = (011) ^ 110 = 101
```

### 格雷码→二进制

```
bin[MSB] = gray[MSB]
bin[i] = gray[i] ^ bin[i+1]
```

## 4. FIFO 空满的具体判断

### 地址扩展法

如果 FIFO 深度为 8（地址线 3 位），就把地址线扩展为 4 位：

```
         高位（判断回绕）    低位（实际地址）
waddr:    0    0    0    0    0    0    0    0
          ↑                    ↑
         满标志看这里              实际 RAM 地址
```

### 空标志（读时钟域判断）

**读指针同步后的写指针 == 读指针** → 空

```
rptr = 0000, sync_wptr = 0000 → 空 ✅
```

### 满标志（写时钟域判断）

**写指针 - 同步后的读指针 == FIFO 深度** → 满

用格雷码的等价判断：

```
格雷码下，wptr 比 rptr 超前 2^(n-1) 即满
```

更直观的二进制判断：
```
满条件：{~wptr[MSB], ~wptr[MSB-1], wptr[MSB-2:0]} == sync_rptr
```

举例（深度 8，4 位地址）：
```
wptr = 1000（地址 0，已回绕一次）
rptr = 0000（地址 0，未回绕）
{~wptr[3], ~wptr[2], wptr[1:0]} = 0100 ≠ rptr → 不满

wptr = 1111（地址 7）
rptr = 0111（地址 7，未回绕）
{~wptr[3], ~wptr[2], wptr[1:0]} = 0011 = rptr → 满 ✅
```

## 6. 面试高频追问

**Q: FIFO 深度怎么算？**
A: 取决于写速率、读速率、突发长度。

```
FIFO 深度 ≥ 写速率 × 突发周期 - 读速率 × 突发周期
```

**Q: 什么时候要用格雷码？**
A: 跨时钟域传多位地址时，用格雷码保证每次只变 1 位。

**Q: 格雷码同步器有延迟，会不会导致空/满判断错误？**
A: 会延迟 2~3 拍。但**只会保守**（可能报告满但实际上没满 / 空但实际上没空），不会漏报 → 功能正确。

**Q: FIFO 深度为什么通常是 2 的幂？**
A: 因为格雷码是循环码，2 的幂才能完美循环。
