## 1. 为什么需要脉冲同步器？

你已经知道**2-flop 同步器**能把信号跨时钟域传过去。

但它有个致命局限：

```
clkA      ▔▔▁▁▔▔▁▁▔▔▁▁▔▔▁▁▔▔
pulseA   ▁▁▁▁▁▁▁▔▁▁▁▁▁▁▁▁▁▁    ← 1个clkA周期宽

clkB      ▔▁▔▁▔▁▔▁▔▁▔▁▔▁▔▁▔▁  ← clkB是clkA的1/3
          ↑采样错过   ↑又错过 → 脉冲丢了！❌
```

**2-flop 同步器只能传"电平"（恒定高或低一段时间），不能传"脉冲"（很窄的跳变）。**

如果要传脉冲（比如 "第1拍开始发送"、"第3拍读数据" 这类单周期使能信号），就需要**脉冲同步器**。

---

## 2. 核心思路：展宽 → 同步 → 恢复

三步走：

```
                 ┌───────────────────────────────────────┐
pulseA（窄）────→│   展宽成电平（toggle flip-flop）       │
                 └───────────────────────────────────────┘
                                   │
                                   ↓ 电平信号（宽度≥1个clkA周期）
                 ┌───────────────────────────────────────┐
                 │   2-flop 同步到 clkB                   │
                 └───────────────────────────────────────┘
                                   │
                                   ↓ 同步后的电平
                 ┌───────────────────────────────────────┐
                 │   边沿检测 → 恢复成脉冲                 │
                 └───────────────────────────────────────┘
                                   │
                                   ↓
                                 pulseB（1个clkB周期宽）
```

---

## 3. 电路结构

```
clkA域                    clkB域
                    ┌────┐        ┌────┐        ┌────┐
pulseA ───┬────────→│FF1 │───────→│FF2 │───────→│FF3 │────┬──→ pulseB
          │         └────┘        └────┘        └────┘    │
          │           ↑              ↑              ↑     │
          │         clkB           clkB           clkB    │
          │                                                │
          └────────────────────────────────────────────────┘
                                  XOR ⨁
```

**具体每步在干什么：**

- **toggle FF（脉冲→电平）**：每次来脉冲，`toggle` 翻转一次（0→1→0→1...）
- **2-flop 同步（跨时钟域）**：`toggle` 电平在 clkB 域打两拍
- **边沿检测（电平→脉冲）**：`sync_2 ^ sync_3`，检测到跳变就输出 1 个 clkB 周期的脉冲

### 时序图

```
pulseA    ▁▁▁▔▁▁▁▁▁▁▁▁▁▁▁▔▁▁▁▁
toggle    ▁▁▁▁▔▔▔▔▔▔▔▔▔▔▁▁▁▁▁▁   ← 每次脉冲来了就翻转

clkB      ▔▁▔▁▔▁▔▁▔▁▔▁▔▁▔▁▔▁
sync_1    xxxx▁xx▔▔▔▔▔▔xx▁▁▁     ← 可能亚稳态，1拍后稳定
sync_2    xxxx▁▁▁▔▔▔▔▔xx▁▁▁     ← 第2拍，已稳定
sync_3    xxxx▁▁▁▁▔▔▔▔xx▁▁▁     ← 第3拍

pulseB    ▁▁▁▁▁▁▁▁▔▔▁▁▁▁▁▁▁▁     ← sync_2 XOR sync_3 = 脉宽1clkB
```

## 4. Verilog 实现

### 完整代码

```verilog
module pulse_sync (
    input  clkA,          // 快时钟域
    input  clkB,          // 慢时钟域
    input  rst_n,
    input  pulseA,        // clkA域的脉冲
    output pulseB          // clkB域的脉冲（宽度=1个clkB周期）
);
    // ---- clkA域：脉冲→电平 ----
    reg toggle;
    always @(posedge clkA or negedge rst_n) begin
        if(!rst_n)
            toggle <= 1'b0;
        else if(pulseA)
            toggle <= ~toggle;    // 来脉冲就翻转
    end

    // ---- clkB域：2-flop同步 + 边沿检测 ----
    reg sync_1, sync_2, sync_3;
    always @(posedge clkB or negedge rst_n) begin
        if(!rst_n) begin
            sync_1 <= 1'b0;
            sync_2 <= 1'b0;
            sync_3 <= 1'b0;
        end
        else begin
            sync_1 <= toggle;     // 把电平同步过来
            sync_2 <= sync_1;
            sync_3 <= sync_2;
        end
    end

    // 边沿检测：检测 sync_2→sync_3 的跳变
    // sync_2 ^ sync_3 = 1 表示电平翻转了 → 说明原来clkA域来了一次脉冲
    assign pulseB = sync_2 ^ sync_3;
endmodule
```

### 关键点理解

```
pulseB = sync_2 ^ sync_3;
```

- 当 `sync_2=0, sync_3=0` → `pulseB=0`
- 当 `sync_2=1, sync_3=0`（电平刚变高）→ `pulseB=1` ✅
- 当 `sync_2=1, sync_3=1` → `pulseB=0`
- 当 `sync_2=0, sync_3=1`（电平刚变低）→ `pulseB=1` ✅

每次 `toggle` 翻转，XOR 都会输出一个高脉冲 → 完美恢复出脉冲！

## 5. 正确性证明

### 问题1：丢失脉冲

```
pulseA ▁▁▔▁▁▁▁▁▁▔▁▁▁▁▁▔▁▁▁▁  ← 3个脉冲
toggle ▁▁▔▔▔▔▔▔▔▔▁▁▁▁▁▔▔▔▔▔  ← 每次翻转，永不失真 ✅
```

`toggle` 是电平信号，**只会在脉冲来时才变**。电平足够宽，2-flop 不会漏采。

### 问题2：恢复出错误数量的脉冲

```
2个脉冲 → toggle翻转2次回到原值 → XOR检测到2次边沿 → 输出2个脉冲 ✅
奇数个脉冲 → toggle翻转奇数次 → XOR检测到奇数次边沿 ✅
```

**结论：不会漏脉冲，不会多脉冲。** ✅✅

### 问题3：多次脉冲间隔太短

如果 `pulseA` 的间隔小于 clkB 的 2 个周期呢？ `toggle` 可能在同步完成前又变了→

```
toggle ▁▁▁▔▔▔▔▁▁▔▔▔▔▁▁▁▔▔  ← 两次翻转靠太近
sync_1 ▁▁▁▁▔▔▔▔▁▁▔▔▔▔▁▁▁  ← 第二次翻转sync_1没来得及采到...
sync_2 ▁▁▁▁▁▔▔▔▔▔▁▁▔▔▔▔▁
```

**限制条件：`pulseA` 的最小间隔必须 ≥ 3×clkB周期**，否则可能丢脉冲。

## 6. 和 2-flop 同步器的关系

| 特性 | 2-flop 同步器 | 脉冲同步器 |
|------|-------------|-----------|
| 适用场景 | 慢→快，电平信号 | 快→慢，脉冲信号 |
| 能传脉冲吗？ | ❌ 可能丢 | ✅ 保证恢复 |
| 延迟 | 2个clk周期 | 3个clk周期 + 半个clkA |
| 面积 | 2个FF/bit | 3个FF + 1个XOR（不管位宽） |
| 输入限制 | 无 | pulse间隔≥3clkB周期 |

## 7. 你之前的 uart_tx 里用到了吗？

想一想：

在 `uart_tx.v` 中：
- `tx_start` 是一个**脉冲信号**（拉高1拍然后拉低）
- 但在你的实现中，`tx_start` 和 `clk` 是**同一个时钟域**的
- 所以不需要脉冲同步器

但如果是**真正的 UART**（两个芯片通信），发送方的 clk 和接收方的 clk 不一样 → tx 和 rx 就是跨时钟域的！这时候就需要脉冲同步器/异步 FIFO 来处理了。

## 8. 面试常见追问

**Q: 脉冲同步器能传多宽的数据？**
A: 只能传**1 bit 脉冲**。如果传多位数据 → 需要用异步 FIFO。

**Q: 脉冲同步器的最大带宽？**
A: 受 clkB 限制，最快每 3 个 clkB 周期传 1 个脉冲。

**Q: 为什么用 XOR 不用其他逻辑？**
A: XOR 检测的是**边沿**（不管 0→1 还是 1→0）。如果用 AND 只能检测一个方向。

**Q: 如果脉冲同步器的输出再接到一个 2-flop 呢？**
A: 没必要。脉冲同步器的输出已经是 clkB 域的同步信号了。
